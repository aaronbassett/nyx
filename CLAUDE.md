## Phase 2 foundation — carried decisions & gotchas
- **Server = Fastify 5 + `@fastify/websocket`** (orchestrator choice; no D-number decided a framework) — every server task inherits it. `buildServer({config, db, mcp, wsHandler?})` is dependency-injected + side-effect-free; `index.ts` owns config-load/fail-fast (DS-003) + `listen`; WS route is `/ws`.
- **`@nyx/protocol`** is the shared zod source of truth for WS events + REST DTOs. Import it via `@nyx/protocol@workspace:*` (it uses source-pointing `exports` — JIT internal-package pattern, no build step). New app tsconfigs must drop the base `rootDir:"src"` (it breaks `tests/`, TS6059) and set `noEmit`; React/Vite apps also need `jsx:"react-jsx"` + `moduleResolution:"bundler"` + DOM libs + the `@/*` alias.
- **Commit subjects MUST start lowercase** (commitlint `subject-case` rejects sentence/start/pascal/upper-case); header ≤72 chars; never `--no-verify`.
- **Before `git push`, run the repo-wide CI gates locally**: `pnpm lint && pnpm format:check && pnpm typecheck && pnpm test`. The husky pre-commit only checks *staged* files, so full-repo eslint/prettier drift stays hidden until CI.
- **Deterministic tests** (constitution IV) use injectable deps — `SessionStore`, the MCP `sessionFactory`, `Queryable` stubs — so integration tests boot the real server with no external Postgres; live-DB checks gate on `DATABASE_URL`; empty packages run `vitest run --passWithNoTests`.

## Phase 3 (US5 auth) — carried decisions & gotchas
- **SIWE auth verification recipe (constitution I — verified from installed types + execution, not memory):** BIP-340 Schnorr via `@midnight-ntwrk/ledger-v8@8.1.0` `verifySignature(vk, bytes, sig)`. Key↔address binding = the unshielded `address` must equal SHA-256(verifyingKey) — compare `addressFromKey(vk)` (32-byte hex; the `.d.ts` "35-byte/prefix" comments are STALE) against the Bech32m-decoded hex (`@midnight-ntwrk/wallet-sdk-address-format@3.1.2`, decode with the address's OWN network or it throws). This is why the verify DTO carries `verifyingKey`. `ledger-v8` is ESM with sync WASM init — loads under Node and vitest with no async `init()`; generate synthetic Schnorr keypairs with `sampleSigningKey()`/`signatureVerifyingKey()`/`signData()` for unit tests.
- **Nonce single-use** = atomic SQL compare-and-swap burn (`UPDATE … WHERE consumed_at IS NULL AND expires_at > now() RETURNING`), burned on ANY attempt (FR-039), inside the same transaction as verify + account-create + session-issue (burn persists on failure; account/session commit only on success). DB clock decides expiry, never the process clock.
- **`SessionAuthStore`** is a write-capable **superset** of the read-only `SessionStore` (issue/slide/revoke) — never add required members to the read-only interface or you break the WS layer's in-memory test doubles. `buildServer` duck-types `db.transaction` to attach a `PgSessionAuthStore`; a transaction-less readiness stub skips auth registration.
- **Any app that consumes `@nyx/protocol` DTOs must declare it as a dependency** (`workspace:*`) — `apps/web` had none. Import **type-only** in the web app so no zod enters the bundle.
- **commitlint also rejects an uppercase/ACRONYM leading word** (e.g. `SIWE …`, `GET …`) as upper/pascal-case — lead the subject with a lowercase word (`add …`, `wire …`).
- **⚠️ US5 owner-gated (not "done" until cleared):** (1) the real-Lace `signData` prefix/byte round-trip (`reconstructSignedBytes`, `"midnight_signed_message:"+len+":"‖payload`) is proven only for internal consistency with a synthetic key — Lace is closed-source; needs an empirical round-trip on a live Lace browser (= the US5 Independent Test). (2) Server-side domain-binding is NOT yet enforced — add an expected-domain assertion when the SIWE message format is finalized. (3) minor: `/auth/nonce` has no issuance rate limit (nonce-table flood; self-cleans via short TTL).

## Phase 4 (US7 persistence) — carried decisions & gotchas
- **The store pattern is now standard** (auth + projects): an interface + a `Pg*` impl + an in-memory double modelling the tables with an injected clock; deterministic behavioral tests via `app.inject()` + the in-memory store; a `pg-*.test.ts` gated on `DATABASE_URL` for the real SQL. **All store failures are promise REJECTIONS, never sync throws** (`Pg*` methods `async`; in-memory returns `Promise.reject`) so `.rejects` and route handlers share one channel.
- **Version model = project-wide monotonic.** A commit (turn batch OR one user edit) = one `db.transaction`: `projects … FOR UPDATE` → `MAX(version)+1` from `project_file_versions` → per-path UPSERT `project_files` + INSERT `project_file_versions`. A rolled-back commit does NOT consume the version (SC-026). Manifest = `project_files` → `(path, contentHash)` ORDER BY path; content hash = deterministic SHA-256 (`node:crypto`).
- **Ownership denies 404 (not 403)** — missing AND not-owned both 404 so existence never leaks (SC-027). Named store errors map to statuses in one handler (413 caps, 409 count-quota, 410 restore-expired, 404 not-found). Size checks use `Buffer.byteLength`. Untrusted uuid `:id` → catch Postgres `22P02` → treat as not-found (never 500).
- **Route auth is shared:** build `requireSession` once in `buildServer` from the resolved auth store and pass it to route groups; a project route group registers only when the auth store exists. Route tests mint a real session via the in-memory auth store.
- **⚠️ Soft-delete cascade seams are no-ops** (`projects/lifecycle.ts`): contract teardown `TODO(T158)`, R2 prefix cleanup, D40 session termination — a deleted project is soft-deleted but its contract/R2/session are NOT torn down until later stories fill the injectable seams. The 30-day recovery window has no config tunable yet (`DEFAULT_DELETION_RECOVERY_DAYS`).

## Phase 5 (US2 compile pipeline) — carried decisions & gotchas
- **The real `../compact-mcp` is a LOCAL compile/analyze toolchain over a workspace (Rust/rmcp)** — `compile` (skip_zk toggle → local `target_dir`), `diagnostics`, `versions`, etc. It has **no R2, no upload, no urlPrefix** — the US2 spec's "MCP holds R2 creds + uploads content-hashed artifacts" was WRONG. Always review `../compact-mcp` source for the real toolchain contract (constitution I), never the spec's assumptions.
- **Resolution — the Compile Service (glue) layer** (`infra/compile-service/API.md`, owner-approved): an HTTP service between Nyx and compact-mcp that holds the ONLY R2 write creds (constitution III), drives the MCP, uploads content-hashed artifacts, and exposes `/v1/check`, `/v1/compile`+poll, `/v1/version`. Nyx consumes it (bearer `COMPILE_SERVICE_TOKEN`) and only READS R2. A SEPARATE agent builds the service; Nyx is built + tested against the contract (mocked). Prefix = `<projectId>/<sourceHash>/`, sourceHash folds compilerVersion+flags (reuse correctness, SC-006).
- **External-service client pattern:** injectable `{fetch, baseUrl, token}` (like `wallet/auth.ts`); a **compile failure is DATA** (`ok:false`/`status:failed`+diagnostics), never a throw; long ops = submit→poll with injectable delay+clock+bounded maxWait (FR-016, no silent timeout/loop). **Verify-before-announce:** GET manifest.json → zod-validate → HEAD every file → emit `artifacts:ready` once; incomplete/stale prefix → reopen guidance, never announced.
- **Web artifact reads** use `mode:"cors"` (R3 COEP-exempt) + `credentials:"omit"` (no cookie on the public R2 read domain).
- **Two "manifest" concepts** — VFS `(path,contentHash)[]` (D38, `@nyx/protocol`) vs R2 `manifest.json` `{sourceHash,compilerVersion,circuits,files[]}` (§5, local web types). Don't conflate.
- **⚠️ US1 wiring seam:** `COMPILE_SERVICE_TOKEN` + `COMPILE_SERVICE_URL` are NOT in `config/schema.ts` yet (US2 has no consumer — orchestrator takes injected deps). US1 must add them (token is a server-only SECRET like DEPLOY_KEY, never client-routed) and construct `HttpCompileClient`. Real Compile Service + R2 integration is **owner-gated**.
