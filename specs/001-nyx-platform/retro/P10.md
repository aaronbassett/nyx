- **Teardown = OFF-CHAIN bookkeeping only** (protocol-forced): registry status→`superseded`/`inactive`; config.ts chokepoint stops resolving `VITE_CONTRACT_ADDRESS` to it (D10); tear down the indexer subscription; GC R2/private-state. The US7 `projects/lifecycle.ts` contract-teardown cascade seam (T054 stub) = **documented NO-OP BY DESIGN** — nothing on-chain to do. Do NOT build an on-chain delete path. **Redeploy supersede = a registry status flip, not an on-chain action.**

### Fixed
- **C1 (CRITICAL double-spend/hung-spinner):** `runDeploy` had NO try/catch around executor/registry calls → a throw (executor stub, transient DB fault) rejects → `router.dispatch` swallows it → no terminal `deploy:status{failed}` (hung spinner); a `recordDeploy` throw AFTER finality → on-chain contract unrecorded → retry (new requestId) double-deploys. The "exactly-once rests on registry idempotency" claim was FALSE (no tx_ref/address unique constraint). **Fix:** pipeline backstop (any throw → terminal `failed` + retriable result, never rejects) + bounded record-retry after finality + **migration 0003 tx_ref UNIQUE index + `recordDeploy` 23505→existing** (structural idempotency, mirrors ledger `deposit_credit`) + memo set before emit.
- **H2 stale-build (HIGH):** queued-during-turn deploys shipped PRE-turn artifacts (build captured at request time; the turn it waits behind mutates the build). **Fix:** re-fetch `getLatestGreenBuild` INSIDE `run` (post-idle), re-reject if null.
- **H2 SC-031 audit blind spot (HIGH):** the exposure audit didn't scan `index.ts` (the `signingKey: config.secrets.deployKey` line) nor the `signingKey` token. **Fix:** extend scope to `index.ts` + add `signingKey` to the emitted-frame/log forbidden set (teeth-proving self-check).
- **M1 teardown race:** `teardownProject` (pooled, no lock) could miss a concurrently-recorded active row → orphan active on a deleted project. **Fix:** run it in a tx with `projects FOR UPDATE` (same lock as recordDeploy).
- **M2 turn-hang wedge:** a queued deploy held the in-flight flag across the turnGate wait; a never-settling turn wedged all future deploys. **Fix:** bounded `queueTimeoutMs` on the queued wait → release + terminal failed status.

### Flagged, NOT fixed (documented)

- **L1 post-announce reorg** — the pipeline consumes one `awaitFinality` verdict; "no phantom on reorg" (SC-029) rests on the owner-gated executor returning `finalized` strictly past reorg depth — a documented obligation of the real `DeployExecutor.awaitFinality` adapter.
