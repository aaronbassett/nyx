## Decisions / spec + wiring contract (extracted)
### Turn lifecycle (the orchestration US1 owns) — maps to existing primitives
`prompt:submit` (D62 entry) → `LedgerStore.openTurn(projectId)` (status `classifying`) → intent classifier (cheap tier, D25): off-domain → `decline(turnId)` (status flip, NO ledger entries, D25/FR-010) + `turn:message` decline reason; accept → `placeReserve(address, turnId, config.tunables.flatReserveNyxt)` (gated `available≥flatReserve` else `InsufficientAvailableError`, status `reserved`, D34). Then ≤3 verify cycles: each = check-compile (`ArtifactOrchestrator.runTurn`, check) → sub-agents write files → simulator `test:results` → `capTestResults` → `VerifyBudget.recordTestResult(pass)` (`green|retry-allowed|exhausted`, D21 `DEFAULT_MAX_CYCLES=3`). Green = sole trigger → `GreenCompileTrigger.triggerOnGreen` (full compile, at-most-once, D35/FR-029) → done-presentation. Exhaustion (D21/FR-004) = honest summary (diagnostics, WIP kept, suggested next prompt), nothing unverified presented. ALWAYS `settle(address, turnId, actualConsumption)` (reserve_release+settlement in one tx, every non-declined outcome, D34) → emit `turn:settled {turnId, consumed, balance}` + `ledger:update`. Infra unreachable (Tome/mnm/toolchain MCP) = 3 backoff retries then loud failure naming the service (scenario 5), still settles at actual.

## ⚠️ Wiring-wave design decisions (recorded)
- **`verify:run {turnId}` S→C event ADDED** — the server drives the turn but the CLIENT's WebContainer runs the OZ/Vitest simulator (US4); there was no signal to run tests. The coordinator's `awaitTestResults(turnId)` emits `verify:run` then awaits the client's `test:results` via a `TestResultsInbox` (register/deliver + timeout, no-hang). Client-side testrunner reaction is owner-gated.
- **Decline terminal signal:** `declined`/`insufficient-balance` don't settle (D25) but the client needs a turn-end signal to unlock input → the coordinator emits `turn:settled {consumed:"0", balance}` (no ledger entry) after those outcomes; `rejected` (turn active) emits only a locked-input `turn:message`.
- **Per-turn ctx-bound seams:** `checkCompile`/`runFullCompile`/`awaitTestResults` need `ctx.send` → built PER prompt:submit (supervisor constructed per-turn over shared sub-agents/ledger/chat).

### Fixed round 1 — turn-loop (2 bugs)
- **CRITICAL double-billing:** single-active-turn was per-WS-CONNECTION (fresh supervisor per `ConnectionContext`) → a reconnect/2nd tab ran a 2nd concurrent turn on the same project → double reserve+settle. **Fix:** `Map<projectId, ProjectTurnState{supervisor, liveCtx}>` — one supervisor per project (D24 lock now spans connections); deferred `liveCtx` closures route in-flight frames to the post-takeover socket.
- **HIGH locked-input + stuck-reserve:** `infra-failed` at reserve-time emitted no `turn:settled` (input locked forever); a settle/appendChat throw was swallowed by `router.dispatch.catch`. **Fix:** `infra-failed{settled}` discriminator, `settleWithRetry`+`SettleFailureError`, `emitTerminalSignal` unlock for never-settled, a loud-logging catch-all backstop.

### Fixed round 2 — the per-project refactor EXPOSED a CRITICAL cross-account hijack
- Per-project state captured `accountAddress` once + swapped `liveCtx` to any newer connection with NO account check, and `runPromptTurn` dispatched on client `payload.projectId` unvalidated, while `authorizeProject` defaulted allow-all (known-open US7 seam). → any attacker could `prompt:submit` a victim's projectId, receive the victim's frames + spend the victim's NYXT. **Fix (3 defenses):** (1) `createProjectAuthorizer(projectStore)` (fail-closed ownership on the unshielded address, D43) WIRED into `createWsHandler({authorizeProject})` in `index.ts`; (2) coordinator rejects `payload.projectId !== ctx.projectId` before any state/ledger touch, keying all per-project state off the ownership-checked `ctx.projectId`; (3) `safeEmit` (dead socket = no-op → turn still settles, reserve released).
- **⚖️ Note (constitution VIII):** US1 had to WIRE the US7 `authorizeProject` ownership seam — its per-project turn state made the previously-tracked "ownership out of scope" gap money-critical. Recorded, not silent.
