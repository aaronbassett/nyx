- **`@midnight-ntwrk/dapp-connector-api@4.0.1`** (latest stable; beta 4.1.0-beta.1 exists — stay on stable). Verified from the actual published `.d.ts`, not memory. The connector v4 `WalletConnectedAPI` provides everything US5 needs:
  - `getUnshieldedAddress(): Promise<{ unshieldedAddress: string }>` — Bech32m; THE account key (D43).
  - `signData(data: string, { encoding: 'hex'|'base64'|'text', keyType: 'unshielded' }): Promise<{ data, signature, verifyingKey }>` — SIWE signing primitive. Wallet prepends a domain-separation prefix. Use `encoding: 'text'` for the SIWE message.
  - `getConnectionStatus(): Promise<{ status:'connected', networkId } | { status:'disconnected' }>` — drives the wrong-network state.

- `@midnight-ntwrk/wallet-sdk-address-format@3.0.0` (POC) — Bech32m codecs (`MidnightBech32m.parse`, per-key codecs) — candidate for the verifyingKey↔unshielded-address binding on the server.

## Patterns & Code
<!-- Reusable patterns, helpful code snippets, architectural insights -->
- **SIWE-for-Midnight flow (client)**: `connect('<networkId>')` → `getUnshieldedAddress()` → GET nonce → `signData(domainBoundMessage, { encoding:'text', keyType:'unshielded' })` → POST `{ address, message, signature, verifyingKey }` to `/auth/verify`. Connector detection ports from the verified POC `pocs/lace-proving/src/midnight/connector.ts` (window.midnight = UUID→InitialAPI{rdns,name,icon,apiVersion,connect}; v4 = has `connect`, legacy = has `enable`; pick Lace by rdns `io.lace.wallet`).
- **Server /auth/verify recipe — VERIFIED from source** (midnight-ledger `ledger-8` + midnight-wallet, exact lines cited in the T035 dispatch / source-investigator report):
  - Signature scheme = **BIP-340 Schnorr / secp256k1** (`k256::schnorr`). Connector `Signature.scheme` defaults to `schnorr_bip340`.
  - Verify with `verifySignature(verifyingKey: string, data: Uint8Array, signature: string): boolean` from **`@midnight-ntwrk/ledger-v8`** (`8.1.0`).
  - **Prefix (spec `SPECIFICATION.md:373`)**: the wallet signs `UTF8("midnight_signed_message:" + byteLength + ":") ‖ payloadBytes`, where payloadBytes = UTF-8 of the message for `encoding:'text'`. The server reconstructs these exact bytes for the `data` arg. ⚠️ FLAGGED GAP: pre-/post-normalization size counting and whether `Signature.data` echoes raw vs prefixed is not 100% pinned by spec + Lace is closed-source → confirm with an **empirical round-trip against a real Lace `signData` response** before trusting (owner-gated; the US5 Independent Test needs a Lace browser). Unit tests can prove the internal logic with a synthetic Schnorr keypair (`ledger-v8` also exports `signData`/`signatureVerifyingKey`).
  - **Key↔address binding**: `unshieldedAddress = SHA-256(verifyingKey.binary)` (`UserAddress::from(VerifyingKey) = persistent_hash(vk)`, persistent_hash = SHA-256). Server: `addressFromKey(verifyingKey)` (ledger-v8) → hex, vs `MidnightBech32m.parse(addr).decode(UnshieldedAddress, networkId).hexString` (**`@midnight-ntwrk/wallet-sdk-address-format@3.1.2`**) → hex; MUST match or reject (prevents key substitution → auth bypass, constitution III).
  - Order: parse+validate request → check nonce unburned+unexpired → **burn nonce (on ANY attempt, FR-039)** → verify signature → verify key↔address → account auto-create (D43) → establish session.
  - `@nyx/protocol` `AuthVerifyRequestSchema` must gain **`verifyingKey`** (the address is a hash of the key, so the server needs the key itself; `{address, message, signature}` alone is insufficient). Both apps consume this DTO — update once, in the protocol package.
  - **ledger-v8 init gotcha**: ESM-only (`"type":"module"`), self-initializes WASM synchronously at import (no async `init()`); our server is NodeNext ESM so a plain `import` works.

## Server auth impl (T033/T035/T036) — verified findings & gotchas
- **ledger-v8 `.d.ts` size comments are STALE.** Runtime `SignatureVerifyingKey`, `addressFromKey(vk)`, and `UnshieldedAddress.hexString` are all **32-byte hex, no 3-byte version prefix** (docs say 35-byte/prefixed). Empirically `addressFromKey(vk) === SHA-256(vk_bytes)` — the key↔address binding is a **direct 32-byte-hex compare** (full vk bytes, no stripping). Confirms the retro recipe.
- **`Bech32mCodec.decode(tclass, networkId)` THROWS on a network mismatch.** Decode the submitted address with its OWN `MidnightBech32m.parse(addr).network`, not a hard-coded id, or a well-formed foreign-network address raises instead of comparing → binding is network-agnostic (wrong-network is the client's four-state concern).
- **ledger-v8 loads fine under vitest** (node export condition → sync `readFileSync` + `WebAssembly.Instance`, no async init, no vitest config). `signData(signingKey, bytes)` takes the **signing (private)** key + signs RAW bytes (no internal prefix); generate synthetic keypairs with `sampleSigningKey()` → `signatureVerifyingKey(sk)`. All malformed inputs throw **catchable** JS errors → `try/catch → false` is a sufficient DoS guard.
- **Fastify async preHandler + `no-misused-promises`:** type the hook `preHandlerAsyncHookHandler` (not `preHandlerHookHandler`, whose void-returning overload trips the rule). `cookie@2.0.1` exports `stringifySetCookie`/`parseCookie` (NOT `serialize`/`parse`).
- **DI seam for auth store:** `buildServer` duck-types `db.transaction` (`isTransactional`) — a real `Db` gets a `PgSessionAuthStore`; the readiness-only stub `Queryable` in foundation.test.ts skips auth registration. Endpoint/middleware tests inject an in-memory `SessionAuthStore` with a clock + call-counters (proves SC-019: resume calls `slide` only, never `issue`).
- **SameSite=Lax** chosen (not Strict): blocks the cookie on cross-site POST (CSRF; all auth mutations are POST) while allowing top-level cross-site GET into the app.

## ⚠️ OPEN GAPS for T039 / owner-gated pass (do NOT forget)
- **Server-side DOMAIN binding is NOT yet enforced.** `/auth/verify` extracts+burns the `Nonce:` line and verifies the signature, but does NOT assert the signed message's domain matches Nyx's expected domain. Scenario 7's "bound to our domain" currently rests only on the nonce being server-issued + single-use + short-TTL. **T039 fixes the SIWE message format (domain/issued-at/statement lines); at that point add a server-side expected-domain assertion (+ a config value for the domain) alongside the nonce check.** Cross-site signature-replay defence-in-depth.
- **Real-Lace prefix round-trip** (`reconstructSignedBytes`) — internal logic proven with a synthetic ledger-v8 keypair; the byte-identical match to Lace's actual `signData` output is UNVERIFIED and owner-gated (needs a live Lace browser). Marked `⚠️ OWNER-GATED` in `verify.ts`.
- **`AuthVerifyRequest` gained `verifyingKey`** — sync `contracts/http-api.md` + `discovery/SPEC.md` `/auth/verify` line to `{ address, signature, message, verifyingKey }` (done at close-out).

## SC-017 measurement (T041, local)
- **`/auth/verify` p95 = 0.408ms** (mean 0.307, p99 1.115, max 1.453) over 350 sampled requests via `app.inject()` with the in-memory store (real WASM Schnorr, no live Postgres). Raw crypto (`verifySignature` + key↔address binding) p95 = 0.235ms. **SC-017 (≤500ms p95) PASSES ~1200× under budget.** The WASM verify is the only heavy compute (~0.2ms); a real-DB transaction (nonce CAS + account upsert + session insert) adds ~tens of ms at most on localhost — still ~450ms headroom. Measured with a throwaway `apps/server/_bench_sc017.ts` (deleted).

## Web ← protocol wiring (T039/T040)
- **`apps/web` had NO `@nyx/protocol` dependency** — it was linked only into `apps/server`. Consuming the shared DTOs required adding `"@nyx/protocol": "workspace:*"` to `apps/web/package.json` (matches the server pattern; no external dep). Imported **type-only**, so no zod enters the web bundle (build JS chunk hash unchanged). Worth pre-checking cross-package availability before scoping a web task as "no installs / consume protocol".
- **Client sends `signed.data`** (the wallet-echoed signed text) as `message`, not the locally-built string — the server then reconstructs its prefix over the authoritative signed bytes. Identical for `encoding:"text"`, but the safer seam if a wallet ever normalises.
