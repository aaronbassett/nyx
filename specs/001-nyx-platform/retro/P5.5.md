# Phase 5.5 Retrospective: Network profiles & local devnet

## What Worked Well
- **Verify-the-toolchain-before-authoring (constitution I) paid off again** (as with US2's compact-mcp review). A read-only `iln`/devnet verification agent found three load-bearing facts BEFORE any compose was written: (1) the devnet is **three separate images**, not one `iln` all-in-one; (2) real version caps; (3) the indexer is Lace-pinned, not remappable — which reversed a design choice pre-commit rather than post-merge.
- **The stock `midnight-tooling:devnet` template *is* our devnet.** Once the indexer decision landed on the conventional 8088, no port remapping was needed — the authoritative template (identity mappings 9944/8088/6300) matched our Lace-compatible profile exactly. Authoring T269 from that template (not memory) kept it correct.
- **Parallel component agents + a verification agent** (server config / web config / preflight / iln-facts) ran cleanly on disjoint file sets; I gated the integration with a repo-wide run before each commit.

## What Didn't Work
- **The design's "remap the indexer to 18088" assumption was wrong.** Lace's Undeployed connector carries an `indexerUri`/`indexerWsUri` (`ServiceUriConfig`), canonically `localhost:8088`, and Lace pins its addresses — remapping would break the wallet's own balance sync (which US6 deposit-funding needs). And it bought no isolation: 9944/6300 are already singular, so two devnets can't coexist regardless. Corrected `18088 → 8088` across config + preflight + tests before the commit.
- **Scoped agent verification missed a repo-wide lint rule.** The web agent declared `NetworkProfile` as a `type` alias; the repo's `@typescript-eslint/consistent-type-definitions` (interface) only tripped on the repo-wide `pnpm lint`, not the agent's scoped checks. Caught by my pre-commit repo-wide gate — reaffirms: always run the full `lint && format:check && typecheck && test` before committing agent output, never trust scoped-green.

## Workarounds & Solutions
- **`infra` was not a workspace package** — only had README/fly.toml/etc. Scaffolding the preflight required making it a real `@nyx/infra` package (package.json + tsconfig + `pnpm-workspace.yaml` entry) so `infra/tests/*.test.ts` runs and it joins the `pnpm -r` gates. It uses vitest defaults (no config file), matching apps/server.
- **Fail-fast port preflight = bind, not connect.** Occupancy is decided by attempting `net.createServer().listen(port, "127.0.0.1")` and treating `EADDRINUSE`/`EACCES` as in-use — never opening a client connection — then `PortsInUseError` lists every occupied port. `devnet:up` runs it before `docker compose up`, so it refuses to reuse a devnet it did not start (closes the deploy-to-a-foreign-chain footgun).

## Patterns & Code
- **`NetworkProfile { id, networkId, nodeUrl, indexerUrl, proofServerUrl }`** is the shared shape, defined as an `interface` on BOTH sides (server `config/network.ts`, web `config.ts`). Server: `resolveNetworkProfile(env: NetworkEnv)` takes a **structural** `NetworkEnv` (not an `Env` import) to avoid a schema↔network cycle; `NYX_NETWORK` enum (default `local-devnet`) + optional per-field overrides (`NYX_NODE_URL`/`NYX_INDEXER_URL`/`NYX_PROOF_SERVER_URL`/`NYX_NETWORK_ID`); `network` lives on `Config`/`PublicConfig` (public, never `secrets`). Web: `VITE_NYX_NETWORK` selects; `EXPECTED_NETWORK_ID` now derives from `NETWORK.networkId`.
- Two built-in profiles: `local-devnet` (default; the working dev target) and `preprod` (public-release target, placeholder endpoints only — real values are TODO(verify), never from memory).
