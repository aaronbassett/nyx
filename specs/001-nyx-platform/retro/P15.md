## Requirements (quote-accurate)
- **FR-047** agent writes = turn-scoped batch commits; USER edits = immediate per-file commit; editor READ-ONLY while a turn is active (D26/D48).
- **FR-078** `.compact` gets Monarch highlighting ported from the LFDT-Minokawa TextMate grammar (D18); other langs use Monaco built-ins. Best-effort — "the compiler owns correctness."
- **FR-079** saves follow the D60 debounced auto-save → immediate per-file commit + VFS write + HMR.
- **FR-080** a `.compact` edit marks verification STALE until the next green cycle; the next turn's agent context includes user-edit diffs.
- **D60** ~1s idle → auto-save (`file:changed` → commit → VFS → HMR); dirty indicator while debouncing; rapid bursts serialize per file.
- **EC-36** unsaved edit at turn start → editor locks + the pending edit is SURFACED for explicit resolution, never dropped. **EC-58** rapid bursts → exactly one commit per file, zero lost. **EC-59** size-cap reject → EC-16 named error in the editor, buffer preserved.
- **SC-045** edit→HMR ≤3s p95. **SC-046** zero lost edits, every save = exactly one commit. **SC-047** Monarch renders the compact-examples corpus without tokenizer errors.
