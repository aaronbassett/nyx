# Phase 2 Retrospective: Foundational

## What Worked Well
<!-- Effective patterns, tools, approaches -->
- **T014 protocol**: zod discriminated unions keyed by `type` gave a clean single source for both apps; `parseEvent(direction, data)` overload narrows correctly. `expectTypeOf` round-trip tests caught inference regressions cheaply.
- Concurrent non-overlapping agents (protocol in packages/, db in apps/server/) parallelised the foundation without file conflicts — only shared pnpm-lock.yaml, which pnpm reconciles.

## Workarounds & Solutions
<!-- Bug fixes, non-obvious solutions, clever workarounds -->
- **T016 db**: raw SQL migrations + a ~200-line typed runner (schema_migrations table, per-migration transactions, pg_advisory_lock against concurrent deploy races) beat pulling in node-pg-migrate — the data-model spec IS literal DDL, so SQL files are the reviewable artifact (constitution V). tsx runs the CLI with NodeNext .js specifiers.
- Append-only ledger_entries enforced by triggers rejecting UPDATE/DELETE/TRUNCATE (not just convention). One-active-deploy via partial unique index WHERE status='active'. Verified against a live throwaway PG15 cluster: 17/17.
- **Concurrent pnpm-add hazard**: two agents running `sfw pnpm add` simultaneously can race pnpm-lock.yaml. Rule adopted: agents that install deps into the SAME package.json run sequentially; cross-app (web vs server) is fine but was still spaced to avoid a simultaneous lock write.

## Batch 2 addendum — T015/T019 (server), T018 (web), + orchestrator fixes

### Decisions / conventions established
- **HTTP/WS framework = Fastify 5 + @fastify/websocket 11** (no D-number decided it; orchestrator call, flagged to owner). WS route at `/ws`; `buildServer({config, db, mcp, wsHandler?})` is DI-testable (no process side effects); `index.ts` owns config-load/fail-fast/listen.
- **Boot config (DS-003)**: pure `loadConfig(env)` throws a named `ConfigValidationError` listing every offender; only `index.ts` prints + `process.exit(1)`, so tests import the validator without exiting. Secrets segregated under `config.secrets`; `publicConfig()` projects them out for anything client-bound (constitution III). Numeric tunables are D47 placeholders, tuned at implementation.
- **D19 model routing** lives in a required `MODEL_ROUTING` JSON env var (shape-validated only, no provider clients yet) — keeps hallucinated model IDs out of the repo (constitution I).

### Patterns worth reusing
- Pure decision fn shared by runtime + tests: `isolationHeadersFor(pathname)` (web) and `loadConfig(env)` (server) — same shape, cheap deterministic tests.
- Injectable factories/stores for DB-free deterministic tests: MCP `sessionFactory`, and the upcoming T022 `SessionStore`. Follow this for every layer that touches an external service.
- D31 MCP contract met with a strict per-call deadline (`Promise.race` + swallow the abandoned settlement) + FIFO semaphore — no silent timeouts, bounded concurrency.
- FR-021 headers: register identical middleware in Vite `configureServer` AND `configurePreviewServer` for dev/preview parity; carve-out match is `path === "/webcontainer/connect" || path.startsWith("/webcontainer/connect/")` (so `/webcontainer/connection` does NOT match).

### Tech-debt / gotchas for later phases
- **`packages/protocol` needed `exports`/`types`** to be importable via NodeNext — FIXED (commit 8b34186, source-pointing JIT pattern). Consumers must add `@nyx/protocol@workspace:*` to their deps. Verified end-to-end with a throwaway server import.
- **Base tsconfig `rootDir:"src"` breaks `tests/`** (TS6059). `apps/web` fixed its own (dropped rootDir/outDir, `noEmit`, kept `include:["src","tests"]` + `vite.config.ts`); `apps/server` fix is folded into T022. Every future app package inherits this — React apps also need jsx `react-jsx`, `moduleResolution:"bundler"`, DOM libs, and the `@/*` alias in tsconfig + vite.
- **Version discipline (constitution I) is load-bearing**: this registry snapshot returns very high "latest" versions (react 19.2.7, vite 8.1.4, vitest 4.1.10, fastify 5.10.0, zod 3.25.76, @modelcontextprotocol/sdk 1.29.0). Always `npm view`, and check peer ranges — `apps/web` pins **vitest 4** (Vite 8 peer range); other packages keep root **vitest 3**; pnpm isolates them. RTL v16 needs `@testing-library/dom` installed explicitly.
- **Empty scaffold packages** (`nyxt-vault`, `scaffold`) use `vitest run --passWithNoTests` so the recursive `pnpm -r test` (CI gate) stays green until they get real tests (nyxt-vault ~T117).
- **MCP tool names are unknown-by-design** — source them from the toolchain/Tome/mnm servers at T067/US1, never from memory (constitution I). `call(tool, args)` takes the name from the caller.
- **Lockfile-race discipline held**: dependency-installing agents were run strictly sequentially (server pass → web → T022); no concurrent `sfw pnpm add` against the shared `pnpm-lock.yaml`.
