## Review outcomes — dual (sonnet+opus) + fixes → DONE
**No Critical — FR-070 (never client-side balance math) is bulletproof and PROVEN** (tests feed an event balance ≠ old±amount and assert the server figure wins). bigint decode correct (rejects JSON numbers, never Number()); injection-clean; typed errors; subscription-once + clean unmount. Fixes applied:
- **H1/M1 (reconnect/replay staleness, both reviewers):** `refetch` had no generation guard → an older `GET /ledger` snapshot could resolve after a live event and clobber it (dropped rows + reverted balance); a replayed `ledger:update` deduped the entry but re-applied its stale balance. Fixed via a fetch **generation counter** (drop stale resolutions) + an entry-`id` **ordering cursor** `appliedThrough` (a snapshot/update at/below the cursor never reverts a fresher balance) + **merge-on-load** (union entries, never drop a live row that arrived during a reconnect refetch).
- **M2:** typed `LedgerFetchError` was discarded → 401 showed "check connection"; now `loadErrorMessage` maps 401/403→re-auth, http/malformed→server-error, network→connectivity.
- **Sonnet H1:** EC-53's required "EC-30 explanation" copy added to the pending row. **M3/L6:** empty bordered `<ul>` on the pending-but-no-entries first-run state fixed. **lastConsumed** now rendered (was dead). **formatElapsed** gained hours.
