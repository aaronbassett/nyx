## Ground truth captured (not from memory)
- **Vitest 4.1.10 JSON reporter** (`--reporter=json --outputFile=...`) — Jest-compatible shape:
  - top-level: `success: boolean`, `numTotalTests`, `numPassedTests`, `numFailedTests`, `numPendingTests`, `startTime`.
  - `testResults: [{ name (abs file path), status, startTime, endTime, message, assertionResults: [{ ancestorTitles[], fullName, title, status: "passed"|"failed"|"skipped"|"todo", duration, failureMessages: string[], meta }] }]`.
  - Parser → `test:results` (`@nyx/protocol` `TestResultsPayload`): `pass = success` (or `numFailedTests===0`); `failures = flatten assertionResults where status==="failed"` → `{ name: fullName, message: failureMessages.join("\n") }`.
- **`test:results` DTO already exists** in `packages/protocol/src/events.ts` (`TestResultsPayloadSchema = { turnId, pass, failures: [{name, message}] }`, `TestFailureSchema`). It is a CLIENT→SERVER event (container emits it up the bridge).
- **Container seams** (`apps/web/src/container/`): `WebContainerHandle` (mount/spawn/fs/onServerReady/onError/teardown), `WebContainerProcessHandle` (output: ReadableStream<string>, exit: Promise<number>, kill()), `PreviewBridge` (send/on). Stream is read-once via `getReader()` in try/finally. `streamProcessConsole` (streams.ts) drains+relays console; `runBootPipeline` (boot.ts) is the spawn→drain-finite→exit pattern testrunner mirrors.

## Decisions / notes (exact spec text captured)
- **FR-028**: `test:results` parsed from structured runner output, emitted via process streams within the same turn; `failures[]` carries per-test name + assertion diagnostics.
- **FR-029**: Green = current cycle's suite passes (no mechanical adequacy gate, D41) AND is the SOLE trigger for full artifacts compile + done-presentation (D35).
- **FR-030**: No retries; run > 120s killed and counted as a failing cycle with timeout diagnostics (D42).
- **FR-032**: Per-circuit coverage MUST be measured + reported as telemetry — never a gate (keeps D41 evidence-based).
- **FR-033**: Test failure payloads capped at 32 KB per event (config tunable) with deterministic truncation that ALWAYS preserves per-test name + first assertion message (REV-002).
- **FR-020**: All agent feedback travels via WebContainer process streams read by the host — never network from in-container (R6).
- **D21**: Max 3 compile-test cycles per turn. On exhaustion the turn ends honestly (failing state summarized w/ diagnostics, WIP files kept in VFS, suggested next prompt). Unverified code NEVER presented as done. Exhausted budget is charged (real work).
- **D35**: Verify-loop iterations = fast static-validity check compiles (no key gen); FULL compile (keys/zkir/manifest→R2) runs on green tests; `artifacts:ready` at most once per successful turn.
- **D41**: Green requires only current cycle's suite passes; test quality owned by steering + Review agent (no mechanical gate). Coverage = telemetry only (accepted risk on record: hollow suite can go green).
- **D42**: A failure is a failure (simulator deterministic); >120s → kill + failing cycle + timeout diagnostics (never silent).
- **SC-013**: `test:results` round trip (spawn→event) ≤ 30s p95 for reference counter suite (owner-gated real-run telemetry).
- **SC-014**: Zero flaky verdicts — identical source+suite → identical results across 100 runs (determinism harness).

## Dual review outcome (Sonnet + Opus on the diff, then fix re-review)
- Both reviewers: **no Critical/High correctness defect in the core**; `verify.ts` accounting + at-most-once green latch confirmed race-free by both (synchronous latch set before `await`). Read-once stream, kill-once, truncation surrogate-safety, cycle-accounting off-by-one, determinism — all confirmed clean.
- **Two HIGH bugs found (Sonnet, empirically reproduced) + fixed** in `testrunner.ts`:
  1. post-exit `await drained` was **unbounded** → a wedged output stream (resolves `exit`, never closes) hung `runTestCycle` forever, violating its "never strands a turn" guarantee + D42's 120s ceiling. Fixed with a `drainWithGrace` helper + injectable `drainGraceMs` (default 5000ms) bounding the drain on BOTH paths.
  2. timeout branch didn't flush the drain → `console:*` could arrive AFTER the terminal `test:results`. Fixed: timeout path now `drainWithGrace` before `finish()`; `test:results` is terminal on both paths (best-effort within grace — perfect ordering on a genuinely-wedged stream is impossible without re-introducing the hang; the no-hang guarantee is the correct trade-off).
- **coverage.ts fixes**: snake_case false-positive (`_` is now a word char: `transfer` no longer miscounts `transfer_from`; `mint_token` still matches as a unit); cap now measures the real `{type,payload,ts}` wire envelope (`EVENT_ENVELOPE_RESERVE_BYTES=53`, worst-case ts) so "≤ maxBytes" is per-EVENT; `MIN_TEST_RESULTS_CAP_BYTES=160` floor + `RangeError` guard replaces a silent over-cap.
- Final: testrunner 19 tests, verify 13, coverage 19. Repo gates green: server 183 passed / 18 skipped, web 153 passed, protocol 57, infra 3.

## ⚠️ US1 wiring seam recorded (do NOT lose — both reviewers flagged)
