# Phase 14 — US13: Project handoff — retro

**Goal (spec.md:455):** "take my project home — as an archive or a git clone — so that Nyx never holds my code hostage." Two READ-ONLY export mechanisms (NOT ownership transfer, NOT GitHub push).
**Independent Test (spec.md:461):** for a project with several turns: the archive hash-matches the latest manifest; a token-URL clone yields the synthesized history read-only; revocation kills the URL immediately.
**⚠️ GATE CHECK: NO hard gate on US13** (needs US7, already merged `68848e5`). Safe to build autonomously (unlike US9/T185).

## Requirements (quote-accurate)
- **FR-074** archive = latest committed tree (SOURCE ONLY, D26 exclusions) + a generated README documenting local-run requirements (`VITE_CONTRACT_ADDRESS`, `VITE_ZK_CONFIG_BASE_URL`, artifact/prover config).
- **FR-075** clone access READ-ONLY behind an unguessable, revocable, REGENERABLE token URL; revocation IMMEDIATE (D58).
- **FR-076** repo history SYNTHESIZED from D48 turn/user-edit versions with descriptive commit messages, CACHED per version watermark (D59).
- **FR-077** handoff DISABLED for soft-deleted projects (D49); archives + repos pass a SECRETS SCAN (PRD §16).
- **SC-042** archive contents hash-match the latest manifest 100%. **SC-043** revoked tokens rejected 100% immediately. **SC-044** zero secrets in adversarial scans.
- **EC-55** token brute-force → entropy + rate limit + logged. **EC-56** concurrent clones + new commit → watermark isolation (consistent snapshot). **EC-57** near-empty project → valid repo w/ README, never error.
- Ownership (D43) gates; secrets never in project files by design (D10 chokepoint — `.env.local` is written to the ephemeral VFS, never persisted as source), the scan is a belt-and-suspenders check.

## Library decisions (npm-view-verified, NOT memory)
- **Archive zip = `fflate@0.8.3`** (zero-dep, deterministic `zipSync`; SC-042 checks the FILES inside match the manifest hashes, not zip bytes).
- **Git materialization = `isomorphic-git@1.38.7`** (pure-JS, ESM, node≥14.17; writes real blob/tree/commit objects from the version rows → a real bare repo — avoids hand-rolling git object/SHA-1 format from memory).
- **Secrets scan = custom regex scanner** (no external dep; curated patterns — PEM private keys, `sk-`/`AKIA`/high-entropy tokens, the deploy key + R2 + provider-key shapes; testable + not expected to fire given D10).
- Installed into `@nyx/server` via `sfw pnpm add` (+36 pkgs).

## Seam map
- **DB already primed:** `projects.clone_token` + `projects.clone_materialized_at_version` + `projects_clone_token_key` partial unique index; `project_file_versions (project_id, path, version, content, content_hash, size, author, created_at)`.
- **`ProjectStore`** (`projects/store.ts`): `getManifest` (path,hash), `getFile` (per-path). **GREENFIELD:** a bulk "current files with content" read + a version-history read (for git synthesis). `loadOwned()` (routes.ts) does NOT check `deletedAt` → handoff routes must add it (FR-077).
- **Protocol DTOs stubbed:** `CreateCloneTokenResponseSchema`/`RevokeCloneTokenResponseSchema` (`http.ts:219`); archive + git are binary/git-HTTP (no JSON body). Contract doc `contracts/http-api.md:26`.
- **Test pattern:** in-memory `ProjectStore` double + `bootProjects` harness (`tests/projects/helpers.ts`) → `app.inject()`; real `computeContentHash` so manifests are byte-comparable. New tests → `apps/server/tests/handoff/`.

## ⚠️ PREREQUISITE GAP (flagged — NOT a formal gate, a real finding)
**Agent-turn file writes are NOT committed to Postgres yet** — `ProjectStore.commit()` has ZERO non-test callers. The supervisor emits `file:write` to the client VFS + feeds the compile orchestrator but never persists to `project_files`; `turn/coordinator.ts` only persists chat. So `project_file_versions` would be EMPTY → a real project's archive + synthesized git history would be hollow. **US13 is built + tested against the store + seeded/committed data** (the store's `commit` works + is tested); the real turn-loop→commit wiring is a US1 gap to flag to the owner (build the handoff correctly now; it lights up once file-commit is wired). Also: **no `turn_id` on `project_files`/`project_file_versions`** → D59 descriptive-per-turn commit messages use the version `author` + a version-ordering/timestamp correlation (or a future turn_id column).

## Review outcomes (dual sonnet+opus, security-focused)
Both converged hard on the two load-bearing security controls. **Solid + genuinely tested:** SC-043 immediate revocation (structural — revoke nulls the column, partial-unique-index single lookup, no stale-cache bypass), read-only by construction (no `git-receive-pack` route), 404-not-403 + 410-soft-delete existence-safe gating, redaction discipline (no path echoes a full secret), deterministic git synthesis, the git-fs `settle` promise-probe pattern, filename sanitization, rate-limit-before-DB.

**Fixes applied (fix agent + mine):**
- **C1 (CRITICAL — opus):** the git-CLONE path ran NO secrets scan (`assertNoSecrets` was archive-only) → `git clone` served secrets verbatim, AND exposed the FULL history (a secret overwritten in a later version survives in `git log`). Fix: run `assertNoSecrets` over the full materialization input (all versions) in `materializeRepo` before build; `handleGitHttp` refuses on a finding (symmetric with the archive), no leak.
- **H1 (recall):** scanner missed the platform's own hex key shapes (`R2_SECRET_ACCESS_KEY` 64-hex, `R2_ACCESS_KEY_ID`, `DEPLOY_KEY`, hex tokens) — mixed-alphabet gate never fires on single-case hex + `detectAssignment` matched the secret-word only as a name SUFFIX (so `R2_SECRET_ACCESS_KEY`/`DEPLOY_KEY` slipped). Fix: substring keyword match anywhere in the key + accept hex values + entropy floor lowered from 4.0 (the unreachable log2(16) max).
- **H2 (precision):** bare high-entropy detector flagged lockfile SRI (`sha512-…`), `data:` URIs, base64 fixtures → the FIRST archive of any project with a lockfile 500'd (violates D17 "never hold code hostage"). Fix: exclude SRI/`data:`/lockfile paths; don't flag bare git SHAs / content hashes. Both directions tested.
- **H3:** zip-slip / `..` / `.git` path guard added to `buildArchive` + `materializeRepo` (`UnsafePathError`).
- **M1/M2:** `app.ts` `trustProxy: true` (Fly edge → real client IP for the EC-55 bucket) + pino req-URL token redaction (the clone token was logged verbatim on every clone).
- **L1:** `handleGitHttp` now wraps materialize/serve (the "never throws" contract). **L4:** archive route catches reserved-path/mismatch/unsafe-path → actionable non-leaking status. **L3 (web, mine):** `HandoffProject.deletedAt` `string|null`→`number` (matches the wire).
- **Documented / owner-gated (not fixed):** M5 clone-negotiation race (info/refs vs upload-pack materialize different trees under a mid-clone commit — fails closed with a git error; the fix needs the owner-gated real negotiation flow); M6 archive two-read snapshot (latent until file-commit is wired); M3 DoS ceilings (packfile cache + LRU eviction — real-load hardening); L5 constant-time token (256-bit, not exploitable); L7 CSRF (relies on the US5 session cookie SameSite); L9 regenerate silently invalidates the URL (product UX). The turn-loop→`ProjectStore.commit` file-persistence gap (US1) still means real projects have empty history until wired.
